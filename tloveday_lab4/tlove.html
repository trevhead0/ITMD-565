Section 3

Implement a degress Celsius to degress Fahrenheit converter. Allow the user to enter a celsius degree value in the text box and click convert to see the fahrenheit.

You will need to bind two event handlers to the click event on the two buttons. User enters celsius value in text input. When they click convert button your code should convert the value to fahrenheit and display the results in the results area to the right. Your results should say "xx degress celsius is equal to xx degress fahrenheit" and be in a p tag. Make sure you clear out the text input on a successful conversion. When the user clicks the clear results button you should clear out the results area.

Make sure you test the input for an empty value and alert "No value has been entered!" if the value is empty. Also test for a non-numeric value and alert "A non-numeric value was entered!" if a non-numeric value was provided.

Section 4

In this section you will create a user supplied number of small boxes of a specific color to the results area. The user enters a quantity and chooses a color and you insert that many boxes of that color in the results area. If the user runs it again it will add the the new quantity of colored boxes to the results area. When the user clicks the 'Clear Results' button all the boxes should be removed from the results area.

Make sure you test the quantity input for an empty value and alert "No value has been entered!" if the value is empty. Also test for a non-numeric value and alert "A non-numeric value was entered!" if a non-numeric value was provided.

The boxes should be made from div elements. Each box should have the background color set to the color the user selected. The boxes should use the inline-block css display option so they flow like text instead of one per line like a block element would. Each block should be 60px square and have a 5px margin on all sides.

Students enrolled in the 565 section will need to add the ability to click on any one box to delete it. Make sure you change the css properties so when the mouse is over the box it looks like a pointer to give the user the impression they can click it.

Section 5

Keyboard Events. For this section we want the user to be able to type any key in the input box below and show the key that was pressed in the results area.

You will need to bind an event listener to the text input to listen for the correct keyboard event and then using the event object passed to the event handler determine which key was pressed.

Display the pressed key in the results area using a 'span' element with a font size of 60px and bold. There should only be one key displayed in the results area at any time and it should be the last one pressed.

Students enrolled in the 565 section will need to also make sure the text input area is cleared between each key press. The last key pressed should remain in the results area until another is pressed. You may need to listen for a separate keyboard event for this.
